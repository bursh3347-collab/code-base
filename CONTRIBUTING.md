# 🤝 Contributing to Code Base

Thank you for your interest in contributing! This guide will help you get started.

## 📋 What We're Looking For

### ✅ Great Contributions
- **New code patterns** extracted from high-star (500+) open-source projects
- **Improvements** to existing patterns (better examples, clearer docs)
- **Bug fixes** in code examples
- **Translations** (especially Chinese ↔ English)
- **Comparison tables** between alternative approaches

### ❌ Not Looking For
- Direct code copies without documentation
- Patterns from projects with incompatible licenses (GPL/AGPL)
- Incomplete snippets without usage examples
- Marketing or promotional content

## 📝 Code Snippet Format

Every code snippet MUST follow this format:

```markdown
### Module Name

> One-line description of what this code does

**Source**: [project-name](github-link) (XXk⭐, MIT)
**Why it's good**: What makes this better than alternatives
**What we changed**: Differences from original

```language
// Actual code
```

**How to use**:
```language
// Usage example
```

**Caveats**: Known limitations or edge cases
```

## 🔄 Pull Request Process

1. Fork the repository
2. Create a branch: `feat/module-name-pattern-name`
3. Follow the code snippet format above
4. Update SOURCES.md with the origin project info
5. Submit PR with a clear description

## 📏 Quality Standards

- Every snippet must have a **source link** with stars and license
- Every snippet must have a **usage example** that actually works
- Every snippet must explain **why it's better** than alternatives
- Documentation language: English (Chinese translations welcome as separate files)

## 🏷️ Good First Issues

Look for issues labeled `good-first-issue` for easy entry points.
